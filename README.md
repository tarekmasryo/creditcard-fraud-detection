# Credit Card Fraud Detection — Practical Modeling and Threshold Tuning

A practical fraud detection workflow using time-aware evaluation, calibrated models, and validation-selected decision thresholds.

The project focuses on evaluating fraud models the way they would be used in an operational setting: balancing fraud detection performance against false-positive alert burden and simple cost assumptions.

Case study: `CASE_STUDY.md`

---

## What this repository includes

- Main notebook: `credit-card-fraud-detection.ipynb`
- Case study: `CASE_STUDY.md`
- Dataset schema notes: `docs/schema.md`
- Optional scoring script: `scripts/score_csv.py`
- Generated artifacts directory: `artifacts/`

The notebook trains baseline and tree-based models, calibrates probability scores, selects validation-based thresholds, and exports reusable artifacts for scoring.

---

## Dataset

This project uses the Kaggle Credit Card Fraud Detection dataset.

Expected columns:

- `Time`
- `V1` to `V28`
- `Amount`
- `Class`

`Class` is the target label:

- `0` = normal transaction
- `1` = fraudulent transaction

The dataset is highly imbalanced, so accuracy is not used as the main measure of model quality.

### Local dataset path

Download `creditcard.csv` and place it here:

```text
data/raw/creditcard.csv
```

### Kaggle dataset path

The notebook also supports the common Kaggle input path:

```text
/kaggle/input/creditcardfraud/creditcard.csv
```

## Getting started

### 1. Install dependencies

```bash
python -m venv .venv

# Windows
.\.venv\Scripts\activate

# macOS/Linux
source .venv/bin/activate

pip install -r requirements.txt
```

The notebook pins `scikit-learn`, `imbalanced-learn`, and `xgboost` versions to avoid Kaggle environment compatibility issues.

### 2. Run the notebook

Open and run:

```text
credit-card-fraud-detection.ipynb
```

The notebook will:

- load and validate the dataset
- run focused exploratory analysis
- create simple amount and time-proxy features
- train Logistic Regression, Random Forest, and XGBoost models
- compare lightweight resampling baselines
- calibrate model probabilities
- tune thresholds on validation data
- evaluate final operating points on the held-out test window
- export artifacts to `artifacts/`

---

## Methodology Notes

- Time-ordered train, validation, and test windows reduce look-ahead leakage.
- AUPRC is the primary metric because fraud detection is highly imbalanced.
- ROC-AUC is still reported, but it can look overly optimistic when the negative class dominates.
- Probability calibration supports threshold-based decisions.
- Thresholds are selected on validation data and reported on a held-out test window.
- Cost assumptions are illustrative and should be replaced with business-specific values.
- The resampling section is a lightweight diagnostic baseline, not the sole basis for the final recommendation.

## Score a CSV

After exporting artifacts, you can score a raw CSV with the expected dataset columns:

```bash
python scripts/score_csv.py --csv data/raw/creditcard.csv --out artifacts/scored.csv --model xgb --policy min_cost
```

The output adds:

- `fraud_proba`
- `fraud_pred`

Use this script as a lightweight inference example, not as a full production fraud monitoring service.

## Limitations

- `Time` is a relative seconds-from-start field, not a real calendar timestamp.
- Hour and day-part features are proxy features derived from `Time`.
- `V1` to `V28` are anonymized PCA-transformed features and are not directly interpretable.
- Cost values used in the notebook are illustrative.
- Thresholds and calibration should be revalidated under data drift before production use.
- This repository is an analytical modeling workflow, not a complete production fraud monitoring system.
